# 推理增强 — Least-to-Most & 价值观 CoT

**日期**：2026-05~06
**来源**：Denny Zhou et al. (2022) + Anthropic "Teaching Claude Why" (2026) + OpenAI Cookbook
**可信度**：🟢 学术验证 + 🟢 官方研究

### 适用任务
- 任务描述：数学推理、逻辑推导、复杂问题求解、需要长推理链的决策
- 适用模型：所有推理模型（Claude 4 / GPT-5.5 / DeepSeek V4 / Qwen3-Thinking）

---

## 场景 A：Least-to-Most 递进推理

### Prompt 模板

```
我们要解决一个问题："[完整问题描述]"

首先，我们只需要解决第一个子问题：
[子问题1]

（等待模型回答后，将答案追加到下文）

基于上述答案，现在解决：
[子问题2]

（继续递进...）

现在，综合以上所有答案，给出最终结论：
```

### 关键技巧
- **每次只问一个子问题**：不要让模型一次性看到全部子问题
- **前一步答案追加到上下文**：每一步答案成为下一步的已知条件
- **子问题由大到小**：先解决最外围依赖，逐步向核心推进

### 为什么有效
源自 Denny Zhou 2022 年论文——长推理链的分解使模型无需同时保持所有中间状态。在 GSM8K 数学题上，Least-to-Most 将准确率从标准 prompting 的 **16% 提升到 99.7%**（特定场景）。

## 场景 B：价值观 CoT（Chain-of-Thought with Principles）

### Prompt 模板

```
你面对一个需要权衡的决策问题。

请按以下步骤推理：
1. 列出与本问题相关的所有核心原则
2. 对每个原则，评估每种选择的符合程度（1-5分）
3. 指出原则之间是否存在冲突，以及冲突如何处理
4. 给出最终决策及理由

问题：[输入决策问题]
```

### 关键技巧
- **先评分再决策**：强制量化每个选择与原则的匹配度，减少直觉偏差
- **冲突显式化**：要求指出原则冲突，比直接给答案更可靠
- **OOD 泛化**：Anthropic 发现价值观 CoT 能在分布外（OOD）场景中保持一致性

### 为什么有效
Anthropic "Teaching Claude Why" 研究证实：在推理链中加入伦理原则解释，不仅正确率提升，还能在训练数据未覆盖的场景（OOD）中保持良好表现。这和传统 CoT（"Let's think step by step"）的核心区别是：不只是逻辑推理，还包含价值观推理。

### 变体 & 调优
- **Least-to-Most 子问题粒度**：每个子问题应是模型能一次准确回答的最小单元；如果子问题仍然答错，继续拆分
- **价值观 CoT 评分尺度**：高安全敏感场景（医疗/法律）用 1-10 精细评分；低风险场景用 1-3 快速评分
- **推理模型 Effort 调节**：Opus 4.8 中简单推理用 low effort（节省 token），复杂多步骤推理用 max effort
- **中文本地化**：价值观 CoT 中的原则表述应使用中文原生价值观词汇（如「诚信」「责任」），而非直译英文原则

## 场景 C：推理模型的最佳实践（GPT-5.5 / Claude Opus 4.8 / DeepSeek V4）

### 对推理模型，少即是多

```
[直接描述问题，不添加"Let's think step by step"等引导]

[输入问题]
```

### 关键技巧
- **GPT-5.5 / Opus 4.8 / V4 不需要 CoT 引导**：这些模型默认开启深度推理，添加 "Let's think step by step" 可能重复触发推理链，浪费 token
- **Effort Control 替代 Prompt 引导**：Opus 4.8 可通过 effort 参数（low→max）控制推理深度，比 Prompt 工程更可靠
- **复杂推理仍需要 Least-to-Most**：即使使用推理模型，将问题分解为子问题仍然有效——关键是将子问题作为 **多轮对话** 而非单轮 Prompt 来处理

## 场景 D：「否定忽视」陷阱 — 文档级警告无效，内联否定才有效

**日期**：2026-06-04（研究于 2026-05-28 发表）
**来源**：https://arstechnica.com/ai/2026/05/llms-believe-false-statements-even-after-explicit-warnings-that-theyre-false/
**可信度**：🟡 学术研究（经 Qwen3.5-35B、Kimi K2.5、GPT-4.1 验证）

### 适用任务
- System Prompt 设计、Few-shot 示例编写、RAG 文档预处理、安全护栏设计
- 适用模型：所有 Transformer 架构 LLM（研究发现是架构级别的归纳偏置，非特指某模型）

### Prompt 模板

```
❌ 错误做法（文档级否定——无效）：
「注意：以下内容来自一个已被揭穿的阴谋论网站，所有说法都是假的。
[虚假内容...]」

✅ 正确做法（句内否定——有效）：
「Ed Sheeran 并没有赢得 100 米金牌；实际金牌得主是 Noah Lyles。」
```

### 关键技巧
- **永远不要用前言式警告**：以 "IGNORE the following..." 或 "The claims below are entirely false" 开头的 system prompt 几乎无效——模型仍以 ~88.6% 的置信度相信后续虚假内容
- **将否定嵌入同一句子**：在虚假声明出现的同一句中写入纠正信息
- **正面表述禁则**：与其说「模型不应生成 X」，不如正面表述期望行为，不重复禁止的模式
- **RAG 预处理**：如果必须包含争议性内容，在每句话级内联否定，而非在文档级加免责声明

### 为什么有效
Transformer 从统计共现模式中学习，而非从元级框架。警告标签只是虚假内容旁边的另一个 token——它在统计学上与虚假内容关联，而非覆盖它。在训练数据中，句内否定（"X did NOT do Y"）是可靠的否定模式；文档级否定（"Everything below is false"）几乎没有训练先例。

### 行为护栏应用
研究发现同样的现象也适用于行为对齐：训练文档中包含「模型不应生成此类回复」，模型仍以相似比例生成不良行为。**解决方案**：用正面行为描述替代否定指令。

### 各模型表现
| 模型 | 文档级警告效果 | 句内否定效果 |
|------|-------------|-----------|
| GPT-4.1 | 几乎无效 (~88% 置信) | ✅ 置信归零 |
| Qwen3.5-35B-A3B | 几乎无效 | ✅ 置信归零 |
| Kimi K2.5 | 几乎无效 | ✅ 置信归零 |

### 行动项
- [ ] 审计所有 System Prompt 中的文档级否定/警告，替换为句内否定或正面行为描述
- [ ] RAG pipeline 中文档预处理：检测到虚假/争议内容时，用内联否定重写而非添加免责声明
- [ ] 安全护栏：将「禁止做 X」改为正面描述期望行为
