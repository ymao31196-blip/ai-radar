# 结构化输出 & Few-Shot 分类

**日期**：2026-05-13（Microsoft Learn 更新）
**来源**：Microsoft Learn Prompt Engineering Guide + OpenAI Cookbook
**可信度**：🟢 官方推荐

### 适用任务
- 任务描述：文本分类、信息提取、实体关系抽取、结构化数据输出
- 适用模型：GPT-4+ / Claude 3.5+ / DeepSeek V4 / 通用

---

## 场景 A：Few-Shot 分类

### Prompt 模板

```
System: 将以下新闻标题分类到对应的主题类别。

Examples:
标题: Twins' Correa to use opt-out, test free agency
主题: Baseball

标题: Qatar World Cup to have zones for sobering up
主题: Soccer

标题: Yates: Fantasy football intel for Week 6
主题: Football

标题: [输入标题]
主题:
```

### 关键技巧
- **示例中不包含目标标签名称**：模型从上下文自动推断分类体系，比显式列出标签更自然
- **2-3 个示例通常足够**：质量 > 数量，示例中的推理过程比答案本身更有价值
- **示例语言与输入一致**：中文任务用中文示例，否则模型会模仿示例语言而非指令语言

## 场景 B：原子事实提取

### Prompt 模板

```
提取以下文本中的事实声明，使用格式：
(实体1, 关系, 实体2)

文本：
"[输入文本]"

要求：
- 每个声明必须是原子事实（不包含"并且"、"此外"等连接词）
- 如果信息不足，标注 [信息不足]
- 引用原文中的具体语句作为依据
```

### 关键技巧
- **原子事实（atomic facts）**：每个提取只包含一个 (主体, 关系, 客体) 三元组
- **引用原文**：要求 citations 显著减少幻觉
- **明确「不知道」的格式**：[信息不足] 比强行编造好

## 场景 C：中文术语提取与翻译

### Prompt 模板

```
你是一个专业的中英翻译和术语管理助手。

任务：
1. 识别原文中的专业术语
2. 为每个术语提供标准中文翻译
3. 输出 JSON 格式的术语表

原文：
[输入]

输出格式：
{"terms": [{"source": "英文术语", "translation": "中文翻译", "context": "出现的上下文句子"}]}
```

### 关键技巧
- **JSON 格式 + response_format**：使用 `response_format: { type: "json_object" }` 配合 JSON Schema，减少中文输出歧义
- **术语识别应包含缩写展开**：第一次出现时提供全称，如 "RAG (Retrieval-Augmented Generation)"
- **上下文提供翻译依据**：要求输出"context"字段，让翻译审核者能验证术语在原文中的正确含义

---

### 为什么有效
- **Few-shot** 示例比纯指令描述更精确地传达输出格式和分类边界（Microsoft Learn 验证）
- **原子事实** 将复杂声明拆分为可验证的最小单元——模型对简单任务远比复杂任务准确
- **结构化格式**（括号三元组、JSON）迫使模型遵循约束，减少自由度 → 减少幻觉

### 变体 & 调优
- 分类场景需要更高准确率：增加更多示例（5-8 个），覆盖边界 case
- 中文场景注意：分隔符用 `"""` 或 `---` 而非中英文混合引号，避免被错误解析
- Token 效率：中文场景用表格而非 JSON 输出，每个字段约节省 1-2 token
