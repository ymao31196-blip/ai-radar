# 气象/气候 AI 模型

> 覆盖：FourCastNet、GraphCast、Pangu-Weather、Aurora
> 这些模型用于全球/区域气象预测，是风场降尺度任务的上游输入来源。

---

## 背景：ECMWF 的外部 AI 模型运行情况

截至 2026 年初，ECMWF（欧洲中期天气预报中心）每日运行以下 AI 气象模型，初始场来自 IFS 业务分析，每日 0 和 12 UTC 各跑两次：

- **FourCastNet** (Nvidia)
- **GraphCast** (Google DeepMind)
- **Pangu-Weather** (Huawei)
- **Aurora** (Microsoft Research)

ECMWF 的 AIFS（AI 集成预报系统）逐步成熟后，计划逐步减少对外部模型的依赖。但这些模型的开源权重和推理代码仍然可用，是风场降尺度研究的黄金基线。

## FourCastNet — Nvidia

**类型**：全球气象预报模型
**日期**：2023-03
**来源**：https://arxiv.org/abs/2202.11214
**可信度**：🟢 已发表

### 是什么
基于 AFNO（Adaptive Fourier Neural Operator）的全球天气预报模型。在 ERA5 再分析数据上训练，0.25° 分辨率（约 30km），预测 6 小时间隔的全球大气状态。

### 对风电/降尺度有什么用
- **最轻量的公开气象模型**：单 GPU 可跑，硬件门槛最低
- **FNO 架构的可迁移性**：AFNO 的傅里叶层设计本身就适合降尺度——从粗网格（全球）到细网格（区域）的映射是算子学习的天然场景
- **频域特征提取**：风电场的湍流、尾流等小尺度现象在频域有特殊分布，FNO 类模型的频域操作可能比其他架构更高效

### 关键信息

| 维度 | 内容 |
|------|------|
| 模型架构 | AFNO（Adaptive Fourier Neural Operator） |
| 输入模态 | 全球网格（0.25°, 720×1440） |
| 输出类型 | 全球气象场预测（10m 风速、2m 温度、位势高度等） |
| 预训练数据 | ERA5 再分析数据（1979-2015） |
| 开源状态 | ✅ 开源（Nvidia 官方） |
| 论文 | https://arxiv.org/abs/2202.11214 |
| 代码 | https://github.com/NVIDIA/FourCastNet |

### 适用场景
- 全球 NWP 输出的快速替代（比传统数值模式快 1000x）
- 降尺度研究的输入层（从全球 0.25° 降尺度到 1-3km 风场级）
- 集成预测（Ensemble）的快速成员生成

### 局限性 / 已知问题
- 仅预测地表变量，无垂直层信息
- 预测时效约 7 天，更长 horizon 精度下降明显
- FN 版本更新后计算量有所增加

## GraphCast — Google DeepMind

**类型**：全球气象预报模型
**日期**：2023-11（Science）
**来源**：https://www.science.org/doi/10.1126/science.adi2336
**可信度**：🟢 已发表（Science）

### 是什么
基于图神经网络（GNN）的全球天气预报模型。核心设计是在六面体网格（icosahedral mesh）上运行 GNN 消息传递，支持多变量（包括 37 个垂直层）和多重网格分辨率。在中长期预报中，90% 的目标指标超过 ECMWF HRES 传统数值模式。

### 对风电/降尺度有什么用
- **垂直层信息**：包含 37 层大气垂直层数据，对边界层风速垂直廓线建模极为重要——直接关联风轮高度（~100m）的风速推算
- **图网格天然适配不规则分布**：风场内测风塔、激光雷达的位置是不规则离散点，GNN 的消息传递可以直接映射到这种非网格结构上
- **Science 级别验证**：全局预报精度经严格验证，作为降尺度输入的可靠性最高

### 关键信息

| 维度 | 内容 |
|------|------|
| 模型架构 | GNN（六面体网格消息传递） |
| 输入模态 | 网格 + 垂直层 + 多变量 |
| 输出类型 | 全球气象场（10天预报，0.25°） |
| 预训练数据 | ERA5 再分析数据（1979-2017） |
| 开源状态 | ✅ 开源（DeepMind） |
| 论文 | https://www.science.org/doi/10.1126/science.adi2336 |
| 代码 | https://github.com/google-deepmind/graphcast |

### 适用场景
- 风轮高度（~100m）风速的 NWP 输入生成
- 基于 GNN 的降尺度模型架构参考
- 中长期（3-10天）风功率预测的上游气象输入

### 局限性 / 已知问题
- 推理耗时较长（显存消耗大）
- 图网格构建的预处理代码复杂
- 开源代码使用的 JAX 生态，与 PyTorch 项目集成需额外工作

## Pangu-Weather — Huawei

**类型**：全球气象预报模型
**日期**：2023-07（Nature）
**来源**：https://www.nature.com/articles/s41586-023-06185-3
**可信度**：🟢 已发表（Nature）

### 是什么
华为推出的 3D Earth-Specific Transformer。使用立方体球面表示（cubed sphere）将地球表面投影到 6 个平面上，再堆叠 Transformer 层处理。在 1 小时、3 小时、6 小时、24 小时预测步长上都超过传统数值模式 IFS。

### 对风电/降尺度有什么用
- **Nature 级别验证**：预测精度经过 Nature 同行评审，可靠性有保障
- **多步长训练策略**：不同预测步长使用不同模型，这种策略可直接迁移到风电的多时间尺度预测（15min / 1h / 6h / 24h）
- **3D 地球表示**：立方体球面对边界层和大尺度环流的表示可能优于经纬网格（极区畸变问题）

### 关键信息

| 维度 | 内容 |
|------|------|
| 模型架构 | 3D Earth-Specific Transformer + Cubed Sphere |
| 输入模态 | 全球网格 + 多变量 + 垂直层 |
| 输出类型 | 全球气象场（7天预报） |
| 预训练数据 | ERA5 再分析数据（1979-2018） |
| 开源状态 | ⚠️ 权重开源，部分代码 |
| 论文 | https://www.nature.com/articles/s41586-023-06185-3 |
| 代码 | https://github.com/198808xc/Pangu-Weather |

### 适用场景
- 需要高可靠性气象输入的降尺度研究
- 多时间尺度预测策略参考（不同 horizon 用不同模型）
- 立方体球面表示在区域降尺度中的应用研究

### 局限性 / 已知问题
- 四个预测模型（1h/3h/6h/24h）分别推理，总资源消耗较大
- 权重已开源但官方代码不完整，社区维护版本质量参差不齐

## Aurora — Microsoft Research

**类型**：地球系统基础模型
**日期**：2024-06（Nature 正刊发表）
**来源**：https://www.nature.com/articles/s41586-025-09005-y
**可信度**：🟢 已发表（Nature）

### 是什么
Microsoft Research 开发的地球系统基础模型，基于 Swin Transformer + 3D感知器架构。不同于其他模型仅限天气预报，Aurora 覆盖空气污染、海洋状态、极端天气等多种地球系统变量。支持灵活的分辨率和变量组合输入/输出。

### 对风电/降尺度有什么用
- **多种地球系统变量**：除风速外，还能输出湍流、气溶胶、海洋状态等——对海上风电的浪-风耦合建模很有价值
- **灵活分辨率**：原生支持 0.1°~1° 范围内的不同分辨率输入输出，不需要额外插值——这本身就是降尺度能力
- **小样本微调**：在大规模地球系统数据上预训练后，在特定任务（如区域降尺度）上仅需少量数据即可微调

### 关键信息

| 维度 | 内容 |
|------|------|
| 模型架构 | Swin Transformer v2 + 3D Perceiver |
| 输入模态 | 网格 + 多变量 + 垂向/水平可调分辨率 |
| 输出类型 | 大气/海洋/空气污染多变量预报 |
| 预训练数据 | 百万小时级地球系统数据 |
| 开源状态 | ⚠️ 权重可用，完整训练代码未公开 |
| 论文 | https://www.nature.com/articles/s41586-025-09005-y |
| 代码 | https://github.com/microsoft/Aurora |

### 适用场景
- 海上风电的浪-风-流耦合预测
- 区域降尺度（原生可变分辨率）
- 极端天气（台风、寒潮）下的风功率预测

### 局限性 / 已知问题
- 完整训练代码未公开，只能做推理和微调
- 推理资源需求高（多 GPU）
- 2026年初 ECMWF 已停止日常运行 Aurora（AIFS 替代），但权重仍可用
