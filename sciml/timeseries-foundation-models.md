# 时序预测基础模型

> 覆盖：TimesFM、MOIRAI / MOIRAI 2.0、PatchTST、Lag-Llama
> 这些模型可以直接作为风电功率/风速预测的基线或 backbone，替代传统 LSTM/GRU 方案。

---

## TimesFM — Google Research

**类型**：时序预测基础模型
**日期**：2024-07（ICML 2024）
**来源**：https://arxiv.org/abs/2310.10688
**可信度**：🟢 已发表（ICML 2024）

### 是什么
Google Research 开发的 decoder-only 时序基础模型。在 1000 亿+时间步的跨领域时序数据上预训练，支持零样本预测。核心设计是用 patching 将时序切块后输入 decoder-only transformer，和 LLM 架构一致。

### 对风电/降尺度有什么用
- **零样本风电预测基线**：直接拿历史风速/功率序列输入 TimesFM，无需训练即可给出合理预测，适合作为对比实验的 baseline
- **下游微调**：在自有风电 SCADA 数据上微调，通常比从零训练的 LSTM 收敛更快、样本效率更高
- **概率预测**：原生支持 quantile 输出，直接给出预测区间——风电并网场景的刚需

### 关键信息

| 维度 | 内容 |
|------|------|
| 模型架构 | Decoder-only Transformer + Patching |
| 输入模态 | 单变量/多变量时序 |
| 输出类型 | 点预测 + 分位数（quantile）概率预测 |
| 预训练数据 | 100B+ 时间步，跨领域（Google Trends, Wiki, 交通等） |
| 开源状态 | ✅ 开源（Apache 2.0） |
| 论文 | https://arxiv.org/abs/2310.10688 |
| 代码 | https://github.com/google-research/timesfm |

### 适用场景
- 短期风电功率预测（0-72h）的零样本基线
- 融合 NWP 气象特征的多变量风速预测
- 作为下游微调起点，加速新场站模型训练

### 局限性 / 已知问题
- 预训练数据以通用时序为主，未包含气象/风电领域数据
- 原生设计面向单变量，多变量需额外包装
- 输入长度和预测长度有上限（默认 512 → 128），长 horizon 需迭代预测

## MOIRAI / MOIRAI 2.0 — Salesforce AI Research

**类型**：时序预测基础模型
**日期**：MOIRAI 1.0 (2024-02) → MOIRAI 2.0 (2025-11)
**来源**：https://arxiv.org/abs/2511.11698
**可信度**：🟢 已发表

### 是什么
Salesforce 开发的通用时序基础模型系列。MOIRAI 2.0 在 3600 万个时序序列上预训练，采用分位数预测 + 多 token 预测架构，目前在 GIFT-Eval 排行榜上排名第一（非数据泄露模型类别）。

### 对风电/降尺度有什么用
- **GIFT-Eval 榜首**：跨领域泛化能力经过严格验证，风电预测任务直接收益
- **分位数预测原生**：输出概率分布而非单点，天然适配电力调度对置信区间的需求
- **多 token 预测**：推理效率比单步自回归快 2-4 倍，适合实时/近实时预测场景

### 关键信息

| 维度 | 内容 |
|------|------|
| 模型架构 | Decoder-only Transformer + 多 token 预测 |
| 输入模态 | 单变量/多变量时序（统一 patch 化处理） |
| 输出类型 | 分位数概率预测（quantile forecast） |
| 预训练数据 | 36M 时序序列，覆盖能源、交通、金融、气候等 |
| 开源状态 | ✅ 开源 |
| 论文 | https://arxiv.org/abs/2511.11698 |
| 代码 | https://github.com/SalesforceAIResearch/moirai |

### 适用场景
- 跨场站通用预测模型（一个模型服务多个风场）
- 需要概率预测输出的调度对接场景
- 时序数据量有限的新建风场（零样本启动）

### 局限性 / 已知问题
- 模型大小分为 small/base/large，large 版本推理开销较高
- 纯时序模型，不原生融合 NWP 气象预报特征

## PatchTST — IBM Research

**类型**：时序预测模型（自监督预训练方法）
**日期**：2023-03
**来源**：https://arxiv.org/abs/2211.14730
**可信度**：🟢 已发表（ICLR 2023）

### 是什么
将时序分割为子序列 patch，输入 Transformer 编码器，支持自监督预训练（掩码重建）。PatchTST 证明了 patching 机制在时序预测中的有效性，是后续 TimesFM、MOIRAI 等 decoder-only 模型的技术前置。

### 对风电/降尺度有什么用
- **自监督预训练**：用大量无标签 SCADA 数据预训练，少量有标签数据微调——风场历史数据丰富但标注稀缺的场景非常适合
- **多变量通道独立**：每个变量独立 embedding，不强制假设变量间线性相关，适合风速、风向、温度、气压等多源异构数据
- **作为分类器**：小样本场景下预训练 PatchTST 的表示可用于风场工况分类（湍流强度、切变等）

### 关键信息

| 维度 | 内容 |
|------|------|
| 模型架构 | Transformer Encoder + Patching + 通道独立 |
| 输入模态 | 多变量时序 |
| 输出类型 | 点预测（可扩展分位数） |
| 预训练数据 | 无需外部预训练数据，在自家数据集上自监督 |
| 开源状态 | ✅ 开源 |
| 论文 | https://arxiv.org/abs/2211.14730 |
| 代码 | 官方 + HuggingFace `ibm/patchtst` |

### 适用场景
- 有大量无标签 SCADA 数据、少量标签数据的场景
- 多变量气象特征（风速/风向/温度/密度）联合预测
- 风场工况分类的表示学习

### 局限性 / 已知问题
- 无法零样本预测，每个数据集需单独预训练或从头训练
- 相比 TimesFM/MOIRAI，没有跨领域大型预训练权重

## Lag-Llama

**类型**：时序概率预测基础模型
**日期**：2023-10
**来源**：https://arxiv.org/abs/2310.08278
**可信度**：🟢 已发表

### 是什么
将 LLaMA 架构适配到时序预测领域，核心创新是用 lag 特征（滞后特征）替代绝对时间位置编码，使模型能捕捉多尺度周期性。输出概率分布（Student-t 分布参数化），支持零样本预测和微调。

### 对风电/降尺度有什么用
- **多尺度周期捕捉**：lag 特征天然匹配风电的日周期（昼夜温差驱动）、周周期（天气系统过境）、季节周期（季风）
- **概率分布输出**：Student-t 分布比简单分位数更灵活，对风速的厚尾分布建模更准确
- **轻量级**：模型规模较小，单 GPU 可微调，适合课题组资源有限的情况

### 关键信息

| 维度 | 内容 |
|------|------|
| 模型架构 | Decoder-only Transformer + Lag 特征 |
| 输入模态 | 单变量/多变量时序 |
| 输出类型 | 概率分布（Student-t 参数化） |
| 预训练数据 | 多个开源时序数据集 |
| 开源状态 | ✅ 开源（Apache 2.0） |
| 论文 | https://arxiv.org/abs/2310.08278 |
| 代码 | https://github.com/time-series-foundation-models/lag-llama |

### 适用场景
- 涉及多尺度周期性的风速/功率预测
- 小规模微调实验（单卡可跑）
- 需要概率分布的调度场景

### 局限性 / 已知问题
- 对极长序列（>1000 步）性能下降
- 社区活跃度低于 TimesFM/MOIRAI
