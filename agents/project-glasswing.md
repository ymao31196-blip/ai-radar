# Project Glasswing — AI-for-Cyberdefense 受控发布模式

**日期**：2026-05-22（初始更新）/ 2026-06-02（扩展公告）
**来源**：https://www.anthropic.com/news/expanding-project-glasswing
**可信度**：🟢 官方

### 核心思路
Anthropic 使用尚未公开发布的 **Claude Mythos Preview** 模型（能力超过 Opus），在受控条件下为关键基础设施组织进行大规模代码安全扫描。模式：
1. Mythos 扫描代码库，发现安全漏洞
2. 同一模型生成修复补丁
3. 人类审核 + 披露流程
4. 积累安全防护机制后逐步公开模型能力

**当前阶段**：从 ~50 个试点组织扩展至 **~150 个组织、15+ 个国家**，覆盖电力、水利、医疗、通信、硬件等关键基础设施。

### 架构图

```
Mythos Preview (受限发布)
        ↓
  ┌─────┴─────┐
  ↓           ↓
漏洞发现    补丁生成
  ↓           ↓
  └─────┬─────┘
        ↓
  人工审核 + 披露
        ↓
  积累安全机制 → 逐步公开模型
```

### 解决了什么问题
- **痛点**：AI 驱动的网络攻击可能成倍增加，传统防御跟不上
- **之前怎么做**：人工渗透测试 + 静态分析工具，速度慢、覆盖不全
- **现在怎么做**：用最强 AI 模型主动扫描 + 自动生成补丁，超前防御

### 关键设计决策
- **能力隔离**：Mythos 仅通过受限项目发布，不对外开放 API
- **闭环修复**：不只是发现漏洞，还要生成可用的补丁
- **渐进公开**：在安全机制成熟后再公开发布（预计 6-12 个月内其他公司也会有同等能力）
- **关键基础设施优先**：电力、水利、医疗等直接影响公共安全的领域优先

### 成果
- 早期扫描中已发现 **10,000+ 个高危或严重安全漏洞**
- 当前瓶颈不是发现漏洞，而是验证、披露和修补已有发现

### 适用场景
- ✅ 适合：安全敏感型组织（金融、医疗、基础设施）的代码审计
- ✅ 适合：开源核心依赖的漏洞扫描
- ❌ 不适合：组织规模太小（目前仅限合作方），无法直接使用 Mythos

### 可复用的模式
- **Foundational model + restricted-access deployment**：高风险能力采用受控发布
- **Scan → Fix → Learn**：AI 安全审计的闭环流程
