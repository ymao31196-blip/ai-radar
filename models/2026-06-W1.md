## Claude Opus 4.8

**日期**：2026-05-28
**提供方**：Anthropic
**来源**：https://www.anthropic.com/news/claude-opus-4-8
**可信度**：🟢 官方

### 是什么
- 模型类型：LLM（前沿推理模型）
- 核心亮点：
  - Dynamic Workflows：可在 Claude Code 中并行启动数百个子 Agent，先规划再扇出执行
  - Effort Control：用户可调节思考深度（low/medium/high/extra/max）
  - 常规模式：价格比 Opus 4.7（$15/$75）便宜 3 倍，降至 $5/$25
  - Fast Mode：速度提升 2.5x，定价 $10/$50
  - 诚实度飞跃：代码缺陷遗漏率约为 Opus 4.7 的 1/4
  - Super-Agent Benchmark 上唯一完成所有端到端案例的模型

### 有什么用
- **大规模代码迁移**：Dynamic Workflows 让单个 Agent 协调数百个子 Agent 并行重构，适合全库级迁移
- **高可靠性编码**：代码缺陷遗漏降至上代的 1/4，生产级代码审查可直接用
- **浏览器自动化**：Online-Mind2Web 84%，超过 GPT-5.5，适合 RPA 和端到端测试
- **成本控制**：Effort Control 允许精细调节成本/质量平衡；简单任务用 low effort，复杂推理用 max

### 实测反馈
- **Hebbia**（金融文档）：更好的引用精度和 token 效率，比 Opus 4.7 便宜 61%

## DeepSeek V4 Flash / Pro

**日期**：2026-04-24 预览版上线（近两周无障碍可用，临近旧模型退役节点）
**提供方**：DeepSeek
**来源**：https://api-docs.deepseek.com/quick_start/pricing
**可信度**：🟢 官方

### 是什么
- 模型类型：LLM（MoE 架构）
- 核心亮点：
  - 世界顶级推理性能，Agent 能力大幅提高
  - 1M 上下文窗口 + 384K 最大输出
  - 支持 Thinking Mode（默认开启）、Tool Calls、JSON Output
  - API 兼容 OpenAI 和 Anthropic 格式
  - **价格屠夫**：Flash 输入仅 $0.14/1M tokens，缓存命中 $0.0028

### 有什么用
- **成本敏感场景首选**：Flash 输入价格是 GPT-5.5 的 2.8%，是 Claude Opus 4.8 的 2.8%
- **缓存重度用户**：缓存命中价 $0.0028，相比 OpenAI 缓存价（$0.50/1M）仅为 0.56%，极适合重复 system prompt / 知识库场景
- **长上下文任务**：1M 上下文 + 384K 输出，全库代码审查、长文档分析无需分块
- **Agent 开发**：原生 Tool Calls + Thinking Mode，且推理 token 不占后续上下文，节省多轮成本

| 多模态 | 纯文本 | 纯文本 |

## Gemma 4 12B

**日期**：2026-05（Google I/O 2026 发布）
**提供方**：Google
**来源**：https://blog.google/technology/ai/（I/O 2026 专题）
**可信度**：🟢 官方

### 是什么
- 模型类型：多模态（统一、无编码器）
- 核心亮点：
  - 统一的多模态架构，无需单独的视觉编码器（区别于 Llama 4 等需要独立 vision encoder 的方案）
  - 12B 规模，可在消费级 GPU / Apple Silicon 上运行
  - 同时发布 QAT（量化感知训练）版本，专为移动端和笔记本压缩优化

### 有什么用
- **移动端多模态**：统一架构无需额外视觉编码器，部署更简单，适合 iOS/Android 本地多模态应用
- **比 Llama 4 更轻量**：12B 统一架构 vs Llama 4 的 17B+vision encoder，端侧占用更小
- **离线隐私场景**：照片分析、文档扫描等无需联网的多模态任务

### 关键指标

| 指标 | 数值 |
|------|------|
| 参数规模 | 12B |
| 上下文窗口 | 待确认 |
| 多模态 | ✅ 统一多模态（无编码器） |
| API 可用 | 开源，Ollama v0.30.4+ 支持 |
| 价格 | 免费（开源自部署） |
| 中文能力 | 待验证 |

### 横向对比

| 维度 | Gemma 4 12B | Llama 4 (推测) | Qwen3-8B |
|------|------------|---------------|----------|
| 参数规模 | 12B | 17B+ | 8B |
| 架构 | 统一多模态（无编码器） | 独立 vision encoder | 纯文本为主 |
| 端侧友好度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| 部署门槛 | 消费级 GPU / Apple Silicon | 需要更多 VRAM | 消费级 GPU |

### 实测反馈
- Ollama v0.30.4 已支持，但 v0.30.5 修复了浮点异常崩溃，建议使用最新版本
- 目前社区反馈有限，属于"尝鲜"阶段

- [ ] 适用场景：移动 App 本地多模态、离线文档/照片分析、隐私敏感场景
- [ ] 关注 Ollama 支持稳定性（已知 v0.30.4/5 有崩溃问题）

## GPT-5.5 / 5.4 / 5.3 Instant 系列

**日期**：GPT-5.5 (2026-04-23) / GPT-5.4 (2026-03-05) / GPT-5.3 Instant (2026-03-03)
**提供方**：OpenAI
**来源**：https://openai.com/research
**可信度**：🟢 官方
**状态**：均已发布并可用，近两周以生态配套更新为主（Codex、记忆功能、AWS 上架）

### 是什么
- **GPT-5.5**（2026-04-23）：前沿推理模型，"a new class of intelligence for real work"，面向编程和专业工作
- **GPT-5.4**（2026-03-05）：性价比更高的专业模型
- **GPT-5.4 mini**：最强 mini 模型，支持 coding/computer use/subagents
- **GPT-5.4 nano**：入门级，$0.20/$1.25
- **GPT-5.3 Instant**（2026-03-03）：日常对话优化，幻觉降低 19.7%-26.8%，ChatGPT 默认模型（`gpt-5.3-chat-latest`）

### 有什么用
- **GPT-5.5**：最复杂推理任务，与 Codex 深度集成，适合多步骤自主编程
- **GPT-5.4**：专业工作的性价比之选，输入价比 5.5 低一半
- **GPT-5.4 mini**：需要 coding/Agent 能力但预算有限的场景，$0.75 input
- **GPT-5.3 Instant**：对话/客服场景，幻觉率显著低于前代

## Qwen3-2507

**日期**：2026-05~06（文档更新）
**提供方**：Alibaba / Qwen
**来源**：https://qwen.readthedocs.io/en/latest/
**可信度**：🟢 官方

### 是什么
- 模型类型：LLM（Dense + MoE 多规格）
- 核心亮点：
  - **Instruct-2507**：通用能力显著提升（指令遵循、推理、长尾知识）
  - **Thinking-2507**：推理任务 SOTA，开源 thinking 模型中的最佳表现
  - 256K 上下文可扩展至 1M
  - 单模型内无缝切换 thinking/non-thinking 模式
  - 支持 100+ 语言

### 有什么用
- **开源推理首选**：Qwen3-Thinking-2507 是当前开源模型中推理能力最强的，可替代闭源模型做推理
- **中文 Agent 私有化部署**：原生中文 + 多规格（0.6B~235B），可按需选择规模
- **灵活模式切换**：同一模型既可做快速响应（non-thinking）也可做深度推理（thinking），简化部署

### 关键指标

| 指标 | 数值 |
|------|------|
| 参数规模 | 多规格：0.6B ~ 235B-A22B（MoE） |
| 上下文窗口 | 256K（可扩展至 1M） |
| 多模态 | 纯文本为主 |
| 推理能力 | ✅ thinking/non-thinking 无缝切换 |
| API 可用 | 开源（HuggingFace），Qwen API 可用 |
| 价格 | 开源免费部署；API 定价待确认 |
| 中文能力 | ⭐⭐⭐⭐⭐ 原生中文 |

### 横向对比

| 维度 | Qwen3-Thinking-2507 | DeepSeek V4 Pro | Llama 4 (开源) |
|------|---------------------|-----------------|---------------|
| 推理能力 | ⭐⭐⭐⭐⭐（开源最佳 thinking） | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 中文能力 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| 部署灵活度 | 0.6B~235B 多规格 | 仅 MoE 大模型 | 多规格 |
| 上下文 | 256K 可扩至 1M | 1M | 待确认 |

- Instruct 版本的指令遵循能力比上一代有明显提升
