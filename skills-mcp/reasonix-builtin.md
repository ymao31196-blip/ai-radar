# Reasonix 内置 Skill & MCP

> Reasonix 开箱即用的 Skill 和 MCP Server，无需额外安装即可使用。
> 
> **命名约定**：工具调用中使用下划线（如 `security_review`），斜线命令中使用连字符（如 `/security-review`）。

## [Skill] explore（子智能体）

**日期**：2025-07-01
**类型**：Skill（子智能体）
**来源**：Reasonix 官方内置
**可信度**：🟢 官方
**平台**：Reasonix

### 一句话描述
在隔离子智能体中做只读代码库探索，返回一份精炼的答案。

### 安装方式
无需安装，Reasonix 内置。直接调用 `explore` 工具或 `/explore` 命令。

### 核心能力
- 跨文件宽网搜索（"找出所有调用 X 的地方"）
- 代码架构理解（"Y 在整个项目中是如何工作的"）
- 调查审计（"审计所有 Z 的使用模式"）
- 子智能体隔离运行，不消耗主会话上下文

### 有什么用
> 当需要跨大量文件查找模式、理解某个系统的工作原理、或在动手修改前摸底代码时，用 explore 代替多次 read_file/grep，结果更聚合且不污染上下文窗口。

### 使用示例
```markdown
# 通过工具调用
explore({ task: "找出项目中所有使用 context.Context 的地方，总结传入模式" })

# 通过斜线命令
/explore 这个项目的认证流程是怎么实现的
```

### 限制 & 注意
- 只读，不能修改任何文件
- 子智能体有自己的上下文限制
- 结果会被压缩为一份答案，不保留中间探索过程

### 同类对比
| 工具 | 优势 | 劣势 |
|------|------|------|
| explore | 隔离上下文、自动聚合、广度优先 | 不可见中间过程 |
| 手动 grep + read_file | 精确控制每一步 | 消耗大量主会话上下文 |
| codegraph_mcp | 结构化代码图谱 | 需要 .codegraph/ 索引 |

## [Skill] research（子智能体）

**日期**：2025-07-01
**类型**：Skill（子智能体）
**来源**：Reasonix 官方内置
**可信度**：🟢 官方
**平台**：Reasonix

### 一句话描述
结合 web_fetch + 本地代码阅读，在隔离子智能体中回答研究性问题。

### 安装方式
无需安装，Reasonix 内置。直接调用 `research` 工具或 `/research` 命令。

### 核心能力
- 同时查询外部文档 + 本地代码
- 对比线上最新 API 文档与本地实现
- 查询"X 库支持 Y 吗？"类问题
- 将在线参考与本地代码交叉验证

### 有什么用
> 当需要查阅外部资料并同时对比本地代码时（如"最新的 React 19 API 我们的项目兼容吗？"），一个 research 调用同时搞定两边。

### 使用示例
```markdown
research({ task: "Go 1.22 的 routing pattern 和我们的 mux 实现是否兼容？查官方 release notes 并对比 internal/router/" })
```

### 限制 & 注意
- 子智能体运行，不可见中间步骤
- 依赖 web_fetch 可达性（内网资源不可用）
- 不执行写操作

### 同类对比
| 工具 | 优势 | 劣势 |
|------|------|------|
| research | 外网+本地一步到位 | 受限于 web_fetch 能力 |
| 手动 web_fetch + grep | 完全掌控 | 两次调用、上下文碎片化 |
| Perplexity MCP | 搜索更强 | 不能读本地代码 |

## [Skill] review（子智能体）

**日期**：2025-07-01
**类型**：Skill（子智能体）
**来源**：Reasonix 官方内置
**可信度**：🟢 官方
**平台**：Reasonix

### 一句话描述
审查当前分支的待提交变更，标记正确性、安全性、测试缺失和隐藏行为变更。

### 安装方式
无需安装，Reasonix 内置。直接调用 `review` 工具或 `/review` 命令。

### 核心能力
- 审查 git diff 中的代码变更
- 标记潜在 bug 和逻辑错误
- 识别测试覆盖缺口
- 发现隐藏的行为变化
- 输出逐文件行号的 verdict 报告

### 有什么用
> 提交 PR 前的「第二双眼睛」——自动发现那些容易被忽略的边界条件、忘记更新的调用点、以及重构引入的副作用。

### 使用示例
```markdown
review({ task: "聚焦在 auth 模块的变更上" })
# 或通过斜线命令
/review
```

### 限制 & 注意
- 只读，不自动修改代码
- 审查质量受 diff 大小影响（过大的 diff 可能遗漏）
- 需要 git 仓库上下文

### 同类对比
| 工具 | 优势 | 劣势 |
|------|------|------|
| review (Reasonix) | 深度语义审查、理解意图 | 仅限当前分支 diff |
| GitHub PR Review | 协作友好 | 偏重风格和模式匹配 |
| security-review | 专注安全 | 不关注通用代码质量 |

## [Skill] security-review（子智能体）

**日期**：2025-07-01
**类型**：Skill（子智能体）
**来源**：Reasonix 官方内置
**可信度**：🟢 官方
**平台**：Reasonix

### 一句话描述
安全焦点审查：注入 / 授权 / 密钥泄露 / 反序列化 / 路径穿越 / 加密问题，按严重程度标记。

### 安装方式
无需安装，Reasonix 内置。直接调用 `security_review` 工具或 `/security-review` 命令。

### 核心能力
- SQL/命令/XSS 注入检测
- 授权和认证逻辑审查
- 硬编码密钥/Token 检测
- 不安全反序列化
- 路径穿越漏洞
- 弱加密算法使用
- 按严重程度分类（Critical/High/Medium/Low）

### 有什么用
> 提交涉及用户输入、文件操作、认证逻辑、外部请求的代码前必跑——发现静态分析工具难以捕获的逻辑层安全漏洞。

### 使用示例
```markdown
security_review({ task: "聚焦 token 处理逻辑在 internal/auth/" })
```

### 限制 & 注意
- 只读，不自动修复
- 不替代专业渗透测试
- 关注代码逻辑层安全，不涵盖基础设施

### 同类对比
| 工具 | 优势 | 劣势 |
|------|------|------|
| security-review (Reasonix) | 逻辑层分析、业务上下文 | 不执行动态分析 |
| Semgrep / CodeQL | 规则化扫描、CI 集成 | 缺少业务上下文 |
| Snyk | 依赖漏洞数据库 | 不审查代码逻辑 |

## [Skill] test（内联）

**日期**：2025-07-01
**类型**：Skill（内联）
**来源**：Reasonix 官方内置
**可信度**：🟢 官方
**平台**：Reasonix

### 一句话描述
运行项目测试套件，诊断失败，提出并应用修复，重跑到通过（同一失败最多 2 次尝试）。

### 安装方式
无需安装，Reasonix 内置。直接调用 `run_skill({ name: "test" })` 或 `/test` 命令。

### 核心能力
- 自动检测项目语言和测试框架（Go / npm / pnpm / yarn / pytest / cargo）
- 运行测试并解析输出
- 诊断失败原因
- 自动提出修复方案
- 应用修复后重跑验证

### 有什么用
> 「测试挂了帮我修」的一键闭环——特别适合重构后快速验证、或修复 CI 上那个总是红的测试。

### 使用示例
```markdown
/test
```

### 限制 & 注意
- 同一失败最多 2 次修复尝试（防止死循环）
- 只在父会话中运行（内联），会占用上下文
- 依赖项目有可运行的测试命令

### 同类对比
| 工具 | 优势 | 劣势 |
|------|------|------|
| test (Reasonix) | 自动诊断+修复闭环 | 仅本地运行 |
| CI/CD | 全面、可重复 | 无自动修复 |
| Copilot 自动修复 | IDE 内集成 | 无批量运行能力 |

## [Skill] codegraph（内联）

**日期**：2025-07-01
**类型**：Skill（内联）
**来源**：Reasonix 官方内置
**可信度**：🟢 官方
**平台**：Reasonix

### 一句话描述
利用 CodeGraph 代码知识图谱进行高效代码检索、影响分析和上下文构建。

### 安装方式
无需安装。需要先在项目中运行 `codegraph` 初始化索引（`.codegraph/` 目录）。

### 核心能力
- 符号级代码搜索（按函数/方法/类型/变量名）
- 调用链追踪（谁调用了 X / X 调用了谁）
- 影响分析（修改 X 会影响哪些符号）
- 全文代码探索
- 文件树浏览

### 有什么用
> 比 grep 更快更精准的代码检索——知道一个函数名就能立刻看到它的完整调用链和影响面，重构前必用。

### 使用示例
```markdown
/codegraph 找到所有调用 executeBatch 的地方，分析修改影响
```

### 限制 & 注意
- 必须先初始化 `.codegraph/` 索引
- 索引可能过时（代码更新后需重建）
- 仅支持已索引的语言

### 同类对比
| 工具 | 优势 | 劣势 |
|------|------|------|
| codegraph | 结构化、调用链、影响分析 | 需要预先索引 |
| grep | 零依赖、即时 | 纯文本、无结构 |
| LSP (gopls) | 实时准确 | 不能批量分析调用链 |

## [Skill] init

**日期**：2025-07-01
**类型**：Skill（内联）
**来源**：Reasonix 官方内置
**可信度**：🟢 官方
**平台**：Reasonix

### 一句话描述
初始化或刷新项目 AGENTS.md，分析代码库（结构/构建命令/架构/约定），写入简洁记忆文件供后续会话加载。

### 安装方式
无需安装。在项目根目录执行 `/init`。

### 核心能力
- 自动分析项目结构
- 识别构建系统和测试命令
- 提取架构和编码约定

```markdown
/init
```

### 限制 & 注意
- 会覆盖现有 AGENTS.md（谨慎使用）
- 分析深度受限于项目规模
- 建议初次使用后手动补充项目特定规则

## [MCP Server] codegraph

**日期**：2025-07-01
**类型**：MCP Server
**来源**：Reasonix 官方内置
**可信度**：🟢 官方
**平台**：Reasonix

### 一句话描述
CodeGraph 代码知识图谱 MCP Server，提供结构化代码检索、调用链追踪、影响分析等 8 个工具。

### 安装方式
Reasonix 内置，无需额外配置。使用前需在项目中初始化 `.codegraph/` 索引。

### 核心能力
| 工具 | 功能 |
|------|------|
| `codegraph_search` | 按名称搜索符号（函数/方法/类型/变量） |
| `codegraph_explore` | **首选工具**：通过自然语言或符号名探索代码区 |
| `codegraph_node` | 获取单个符号的完整定义、调用链和源码 |
| `codegraph_impact` | 分析修改一个符号会影响的依赖 |
| `codegraph_callees` | 列出某符号调用了谁 |
| `codegraph_callers` | 列出谁调用了某符号 |
| `codegraph_files` | 按语言/目录浏览索引文件树 |
| `codegraph_status` | 索引健康检查 |

### 有什么用
> Reasonix 最核心的代码理解基础设施——几乎所有代码相关的 Skill 都依赖它来检索代码。比 grep 更智能，比 LSP 更全局。

### 使用示例
```markdown
# 探索认证流程
mcp__codegraph__codegraph_explore({ query: "AuthService loginUser session-manager" })

# 追踪调用链
mcp__codegraph__codegraph_callers({ symbol: "executeBatch" })

# 重构前影响分析
mcp__codegraph__codegraph_impact({ symbol: "UserService.CreateUser" })
```

### 限制 & 注意
- 必须先初始化索引（项目需有 `.codegraph/` 目录）
- 索引是快照，代码更新后需重建
- 结果有数量上限（默认 10-20 条）
- 跨项目查询需传 `projectPath`

### 同类对比
| 工具 | 优势 | 劣势 |
|------|------|------|
| codegraph MCP | 结构化、调用链、影响分析 | 需要预先索引 |
| GitHub Code Search | 零索引、跨仓库 | 无调用链、只文本匹配 |
| Sourcegraph | 企业级、多语言 | 部署复杂、需服务端 |
| LSP (gopls/rust-analyzer) | 实时、IDE 集成 | 单文件、无全局调用分析 |

### 社区评价
- Reasonix 最独特的差异化能力之一
- 相比 Claude Code 的 grep 方案，codegraph 的结构化搜索更精准
- 建议在 CI 中自动更新索引以保持同步

### 行动项
- [ ] 在项目中初始化 `.codegraph/` 索引并验证搜索效果
- [ ] 重构前使用 `codegraph_impact` 做影响分析
